# A Toy Compiler Targeting Linux RISC-V 32I

This is a compiler project that compiles a toy language (defined in `grammar.ebnf`) to RISC-V 32I assembly code that works in a Linux environment.

The language is simple enough that there's

- Only an integer type
- Only stack-allocated memory
- Only one or more function definition at the top level
- No global variable
- No comment
- No support of multiplication and division!

Due to the minimalism, we could write interesting program like

```text
fn multiply ( n : int , m : int ) -> int {
    let product : int := 0 ;
    if ( m == 0 ) { product := 0; }
    elif ( m > 0 ) {
        if ( n > 0 ) { product := multiply ( n , m - 1 ) + n; }
        elif ( n == 0 ) { product := 0; }
        else { product := multiply ( n , m - 1 ) - n; }
    }
    else {
        if ( n > 0 ) { product := multiply ( n , m + 1 ) - n; }
        elif ( n == 0 ) { product := 0; }
        else { product := multiply ( n , m + 1 ) + n; }
    }
    return product;
}
```

Note: I specifically emphasized that this only works for Linux because the compiled code
uses Linux convention like using the `li a7, 93` `ecall` exit sequence

## How to Use it?

After cloning this project,

```console
cargo build --release --target-dir .    # build the project into binary
./release/compile <src_file>            # use the binary to compile src_file
```

As an example, `<src_file>` containing `fn main() -> int { return 0; }` will be compiled to

```riscv
# Generated by a toy compiler from hi.lang
# Target: RISC-V (RV32I)

.text
.global _start
_start:
    call main
    li a7, 93
    ecall
main:
    addi sp, sp, -16
    sw ra, 4(sp)
    sw s1, 8(sp)
    sw s0, 12(sp)
    addi s0, sp, 16
    li a0, 0
    lw ra, -12(s0)
    lw s1, -8(s0)
    lw s0, -4(s0)
    addi sp, sp, 16
    ret
```

How many temporary register is enough?

````text
comp_expr                dest=t0  free={t0,t1,t2}
├── arith_expr (left)    dest=t0  free={t0,t1,t2}
│   ├── atom_expr        dest=t1  free={t0,t1,t2}
│   ├── (+/-)
│   └── atom_expr        dest=t2  free={t0,t2}
├── (==/>)
└── arith_expr (right)   dest=t1  free={t1,t2}
    ├── atom_expr        dest=t1  free={t1,t2}
    ├── (+/-)
    └── atom_expr        dest=t2  free={t2}  ← last free temp

Legend:
  dest  — the register this node writes its result into
  free  — registers still available for this node and its children to use

Rule: each right sibling consumes one register from the free pool,
````
